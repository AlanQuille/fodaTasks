Task 1: counts¶

This task was accomplished by writing the counts(input_list) function (where input_list is an argument with data type list) in Python, shown below.

The function uses the unique(list1) function (where list1 is an argument with data type list) to get a list of unique elements from list1 (i.e. if the element is present in list1, it will be present in the unique list only once, irrespective of how many times it is present in list1). This was taken from the website GeeksforGeeks 1.

The unique list is searched and each element is compared with each element of input_list. If there is a match, a dictionary is incremented with the key being equal to the element of the unique list.

The algorithm for unique(list1) is given as follows:

  1. An empty list called return_list is defined.
  2. Each element of list1 (called x) is traversed in a for loop.
  3. In the loop, if x is not in return_list, x is appended to return_list.

When the traversal of list1 is finished, return_list is returned by unique(list1).

The algorithm for counts(input_list) is given as follows:

  1. unique(list1) is called with input_list as the argument list1.
  2. The return value of this call is assigned to unique_list.
  3. Each element of unique_list (called i) is traversed in a for loop.
  4. A value of 0 is assigned to the dictionary output_dict in the loop at key = i.
  5. Each element of input_list (called j) is traversed in another for loop in this loop.
  6. If i equals j, the value of output_dict at key= i is incremented.

When the traversal of unique_list is finished, output_dict is returned by counts(input_list).

The function definition for unique() is given as follows:

In [28]:
# Assigns how many times an entry appears
# in the list argument input_list to a 
# dictionary. The dictionary's keys are 
# the unique elements of input_list
def counts(input_list):

    # The dictionary counts returns
    output_dict = {}
    # Unique elements of input_list
    unique_list = unique(input_list)

    # Traverse each element of unique_list:
    for i in unique_list:
        # the number is initially 0 for each key of output_dict
        output_dict[i] = 0
        # Traverse each element of the original input_list
        for j in input_list:
            # if element of unique_list = element of input_list
            if(i==j):
                # increase the count for each key of output_dict
                output_dict[i] += 1
        
    # Return the dictionary
    return output_dict

Task 2: dicerolls¶

The task is to simulate a dice or die being rolled a number of times. This task was accomplished by writing the dicerolls(n, k) function (where n is an integer representing the number of dice rolls and k is an integer representing the number of die).

The function works by using numpy to generate random integers between 1 and 6 inclusive for k die, summing the result. This process is repeated n times.

The algorithm for the dice_rolls is given as follows:

  1. An empty dictionary called output_dict is defined.
  2. A variable called sum is defined and set to 0.
  3. A for loop with range 0 and n is called.
  4. In the for loop, a function from the random module in the numpy package called randint is called. 2
  5. This function generates a list of randomly generated integers from 1 to 7 called dice_rolls. The list is of size k.
  6. dice_rolls is summed using the sum function from the numpy package. The variable called sum is set to this sum.
  7. An if statement determines if sum is not already in the keys of the dictionary output_dict
  8. If the if statement returns True, than output_dict at key sum is set to 1.
  9. If it returns False, an else statement increments the output_dict at key sum by 1.

The dicerolls function is called with n=1000 and k=2. The result is set to d.

In [5]:
# numpy for numpy array
import numpy as np
# collections for OrdererdDict
import collections
# pyplot for plotting.
import matplotlib.pyplot as plt

# dicerolls function
def dicerolls(k, n):
    
    # output_dict empty dictionary
    output_dict = {}
    # sum variable set to 0
    sum = 0
    
    # for loop from 0 to n
    for i in range(0, n):
        
        # dice_rolls numpy array of random integers from 1 (inclusive) to 7 (not inclusive)
        # to simulate dice roll. This is done k times (number of die=k)
        dice_rolls = np.random.randint(1, 7, size=k)
        
        # sum is set to sum of dice_rolls
        sum = dice_rolls.sum()
        
        # if sum is not in the keys of output_dict
        # set output_dict[sum] to 1.
        # else increment output_dict by 1
        if(sum not in output_dict.keys()):
            output_dict[sum] = 1
        else:
            output_dict[sum] += 1
    
    # output_dict is return value of dice rolls
    return output_dict

Task 3: coin flipping¶

The task is to show that flipping a coin repeatedly and plotting the distribution of the number of heads results in a bell-shaped curve. This is accomplished using the numpy.random.binomial function in the random module in the numpy package. 5

This function is used to generate the sum of 100 random integers between 0 and 1 inclusive, so there is a 50% chance of getting a 0 (heads) and a 50% chance of getting a 1 (tails) (this is identical to a perfectly unbiased coin flip) 6. The sum will add 0 if it is tails (failure) and 1 if it is heads (success) so the sum is identical to the number of heads. This is done 1000 times and is output to a numpy array called coin_flip. Afterwards, the dataset of the number of heads per iterations is plotted using a histogram using the pyplot module from the matplotlib package.

Most of the values are close to the mean of coin_flip (50) and the number of heads decreases the further away the value is from the mean. However, this can be made clearer by plotting the dataset using a kernel density estimate (KDE) plot. KDE plots first estimate the probability density function of a random variable using a given dataset and then plots the resulting PDF. This is accomplished using the seaborn module which has the function kdeplot.

First, a dataset called normal_pdf is generated using the probability density function of the normal distribution 7, the mean, standard deviation and size being equal to the mean, standard deviation and size of coin_flip. Then the KDE graphs of coin_flip are plotted on the same axes of a plot of normal_pdf using the plot function in pyplot to compare the normal distribution with the distribution of coin_flip:

Task 4: Simpson's paradox¶

The task is to demonstate Simpson's paradox 7. This is accomplished by using the normal_multivarate function from the numpy.random module. 8.

Simpson's paradox is the phenomenon whereby relationships, trends and correlations observed for particular sections of data change direction when the data is aggregated. An archetypal example is the phenomenon showing sex bias in graduate school admissions in UC Berkeley in California 9.

When performing a Chi-squared test on the graduate school admissions data in aggregate, it showed that more men then women were admitted and it was not due to random chance, providing evidence of bias in favour of men. However, when the test is performed again for each graduate school, it showed that there was evidence of bias in favour of men. In this particular case, the reason was more women applied to graduate schools which were tougher to enter and men were more likely to apply to schools that were easier to enter. So even though there was actually bias in favour of women at these schools, much more men were being accepted overall so it appeared that there was bias in favour of men overall at UC Berkeley.

This phenomenon applies to continuous datasets as well. This is demonstrates using the multivariate_normal function in the numpy.random module. Imagine an engineering company trying to increase the longevity of a device they make by increasing the density of the material the device is made from, for example.

The company discovers through 1000 experiments that the mean longevity is 10 years, the mean density is 8050 kg/m^3, the standard deviation of the longevity is 5 years and the standard deviation of the density is 50 kg/m^3. They also discover a positive correlation of 0.4 between the density and longevity.

In aggregate, the data is generated like this (the dataset is initially generated by creating a covariance matrix from the correlation matrix to use in multivariate_normal 10)

In [49]:
# calculate covariance matrix from correlation matrix
from statsmodels.stats.moment_helpers import corr2cov
# mean longevity is 10 years
# mean density is 8,050 kg/m^3
mean = [10, 8050]
# the standard deviation of the longevity 
# is 5 years, the density is 50
std = [5, 50]

# the correlation between these two variables is 0.4
# and this correlation matrix reflects this
correlation = [[1, 0.4],[0.4, 1]]

# using corr2cov to generate covariance matrix
cov = corr2cov(correlation, std)

# y = longevity in years, x = density in kg/m3
# from multivariate_normal
y, x = np.random.multivariate_normal(mean, cov, 1000).T

Method A produces a statistically insignificant positive correlation, Method B produces a statistically insignificant negative correlation as does Methods C and D. This is is in constrast to the correlation for the data in aggregate.

This is a hypothetical dataset but if this was real an explanation could be that the different methods used themselves effect the longevity independent of the density. So Method A does not effect the longevity significantly but Method B causes a large jump in longevity, as does Method C and D.

These jumps would not be apparent overall because it looks like increasing the density is what is causing the increase in longevity because of the positive correlation. This would be an example of "correlation does not imply causation" 11.

This demonstrates Simpson's paradox that aggregating the data and produce different correlations, relationships and trends as opposed to dividing the data into its constituent groupings.
